# 小考學習歷程總覽 (Quiz Learning Journey)

本頁面彙整了我在 **CS203A 資料結構** 課程中五次小考的學習精華。內容涵蓋從底層 C 語言記憶體管理到高階圖形演算法的理論與實踐，並記錄了我在錯誤中修正觀念的過程。

---

## 小考單元概覽 (Quiz Summary)

### [Quiz 1：C 語言基礎與複雜度分析](./Quiz1/README.md)

* **核心知識**：
  
    * **動態記憶體分配**：學習使用 `malloc` 、`realloc` 與 `free` 管理程式運作時的記憶體空間。
      
    * **時間複雜度 (Big O)**：理解不同運算階層的效能差異，包含 $O(1)$ 、$O(n)$ 與 $O(n \log n)$。
      
* **關鍵訂正**：
  
    * **安全防護**：體會到在分配記憶體後進行 `NULL` 檢查的重要性，以防止系統崩潰。
      
    * **組合運算**：釐清執行 $n$ 次二分搜尋的總體複雜度應為 $O(n \log n)$，而非單次的 $O(\log n)$。

### [Quiz 2：線性資料結構](./Quiz2/README.md)

* **核心知識**：
  
    * **鏈結串列指標操作**：在單向鏈結串列中移動節點，必須先定位前驅節點 (Predecessor) 以維持串列完整性。
      
    * **堆疊 (Stack)**：遵循 **LIFO** (後進先出) 原則，核心操作包含 `push` 與 `pop`。
      
    * **佇列 (Queue)**：遵循 **FIFO** (先進先出) 原則，核心操作包含 `enqueue` 與 `dequeue`。
      
* **關鍵訂正**：
  
    * **ADT 完整性**：除了基礎增刪，`isEmpty`、`isFull` 與 `peek/front` 也是定義資料結構狀態的重要操作。

### [Quiz 3：Hashing](./Quiz3/README.md)

* **核心知識**：
  
    * **碰撞處理 (Collision)**：理解並比較**開放定址法 (Open Addressing)** 與**鏈結法 (Separate Chaining)** 的物理存放差異。
      
    * **效能分析**：探討負載因子 ($\alpha$) 如何影響性能，以及線性探測引發的初級聚集 (Primary Clustering) 問題。
      
    * **哈希函數**：實作除法取餘法與折疊法 (Folding Method)，觀察資料分佈的均勻度。
      
* **關鍵訂正**：
  
    * **桶 (Bucket) 定義**：釐清桶是哈希表中的基本儲存單位，可直接存放資料或作為鏈結串列的起點。

### [Quiz 4：圖形遍歷與表達式樹](./Quiz4/README.md)
* **核心知識**：
    * **廣度優先搜尋 (BFS)**：利用 **Queue** 進行層級遍歷，並透過 **Visited** 陣列處理環路問題。
    * **表達式樹運算**：運用**中序走訪 (Inorder Traversal)** 將樹結構轉化為人類可讀的算式並求值。
    * **分類邏輯**：精確區分形狀基準 (Complete Binary Tree) 與規則基準 (BST/AVL) 的演化限制。

### [Quiz 5：樹與圖形術語及結構轉換](./Quiz5/README.md)

* **核心知識**：
  
    * **術語體系**：精確定義度數 (Degree)、高度 (Height)、葉節點 (Leaf) 與子樹 (Subtree) 等核心術語。
      
    * **結構特殊化**：掌握將一般多叉樹轉換為二元樹的「**左兒子，右兄弟** (Left-child, Right-sibling)」演算法。
      
* **學習成果**：透過視覺化繪圖，理解如何將非結構化的一般樹轉化為易於操作的標準二元樹。
