## 三、學習要點

透過這次作業，我學到以下重要觀念與技能：

- **什麼是 Repository** : 一個集中管理專案與課程資料的空間。

- **README.md 的重要性** : README 是讓他人快速理解專案內容與用途的說明文件。

- **什麼是 Commit** : 每一次修改都能被清楚保存，有助於回顧與管理學習歷程。

- **GitHub Pages 的用途** : 能快速建立個人網站，作為課程首頁或未來作品集展示平台。


## 四、GitHub 初次建立與使用流程指南

### Step 1：建立 GitHub 帳號

1. 前往 https://github.com

2. 點選 **Sign up**

3. 註冊並完成帳號設定

### Step 2：建立課程儲存庫

1. 點選右上角 **+ → New repository**

2. Repository name：`11401_CS203A`

3. Visibility：`Public / Private`

4. 勾選 **Add a README file**

5. 點選 **Create repository**

### Step 3：編輯 README.md

1. 點選 `README.md`

2. 點擊 **Edit**

3. 修改內容後點選 **Commit changes**

### Step 4：建立個人網站儲存庫

1. 新增儲存庫

2. Repository name：`username.github.io`

3. Visibility：`Public / Private`

4. Create repository

### Step 5：建立 index.html

1. 進入 `username.github.io` 儲存庫

2. 點選 **Add file → Create new file**

3. 檔名輸入：`index.html`

4. 輸入 HTML 內容

5. 點選 **Commit changes**

### Step 6：啟用 GitHub Pages

1. 進入儲存庫 → **Settings**

2. 點選 **Pages**

3. 設定：

- Branch：`main`

- Folder：`/ (root)`

4. 儲存後即可透過以下網址瀏覽：

- https://username.github.io/

### 補充：修改儲存庫可見性（Public / Private）

若儲存庫已建立完成但需要更改可見性：

1. 進入儲存庫 → **Settings**

2. 點選左側 **General**

3. 找到 **Repository visibility**

4. 點選 **Change visibility**

5. 選擇 `Public` 或 `Private`

6. 依照提示確認並儲存
